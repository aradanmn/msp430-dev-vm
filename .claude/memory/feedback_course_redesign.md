Exercises must require genuine problem-solving, not copying register values from tutorials.

**Why:** Scott completed L01–L05 and realized he was mostly transcribing register configurations. The bugs he made were transcription errors (typos), not understanding errors — proof the exercises didn't teach the material. He wants to design embedded systems from scratch.

**How to apply:**
- Tutorials teach concepts, not register recipes. Point to the datasheet for specifics.
- Never hand the student exact register bit patterns in exercises — they should look them up.
- Include debugging exercises (broken code with realistic bugs) in every lesson from L02+.
- Student writes all handheld modules from behavioral specs — no pre-built code.
- Three exercise types: explore (build it), challenge (debug/design), milestone (handheld module).
- If the student can solve an exercise by copying from the tutorial, it's too easy.
